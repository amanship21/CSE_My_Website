Essential Commands

The commands every Linux user needs to know — hover any row to highlight.

📂 File System Navigation
ls -lalist all files with details
cd /pathchange directory
pwdprint working directory
find / -namesearch for files by name
cp -r src dstcopy files recursively
mv old newmove or rename a file
rm -rf dirforce-remove directory ⚠️
🔑 Permissions & Ownership
chmod 755 fileset rwxr-xr-x permissions
chown user:grpchange owner and group
sudo commandrun as superuser
su - userswitch user account
groupsshow current user groups
idshow uid/gid info
umask 022set default file mask
🌐 Networking
ip ashow all interfaces
ping hosttest connectivity
ssh user@hostremote shell login
scp file u@h:secure copy over SSH
curl -I urlfetch HTTP headers
netstat -tulnpshow listening ports
ss -ssocket statistics summary
🖥️ System & Processes
top / htoplive process monitor
ps auxlist all running processes
kill -9 PIDforce-kill a process
df -hdisk usage by filesystem
free -mRAM usage in megabytes
systemctl statuscheck a service status
journalctl -xeview system log errors

Cloud Engineer Roadmap

A clear path from complete beginner to job-ready cloud support engineer.

✓
Phase 01 — Foundation

Linux Basics & Shell

Get comfortable with the terminal. Learn navigation, file management, and basic commands.

02
Phase 02 — Administration

Users, Packages & Services

Manage users, install packages with apt/yum, control services with systemd, and schedule tasks.

03
Phase 03 — Networking

Networking & SSH

Configure networking, set up key-based SSH auth, and understand firewalls with ufw/iptables.

04
Phase 04 — Automation

Bash Scripting

Write scripts to automate real sysadmin tasks. Build your own toolbox of reusable utilities.

05
Phase 05 — Web & Cloud

Nginx, Docker & Cloud CLI

Host web apps with Nginx on your VM, containerize with Docker, and interact with cloud providers via CLI.

06
Phase 06 — Career

Cloud Support Engineer 🎯

Build projects, write documentation, and land your first role in cloud support or DevOps.
